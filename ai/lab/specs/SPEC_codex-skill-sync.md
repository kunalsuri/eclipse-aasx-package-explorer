## 1. Goal

Populate `.codex/skills/` with the union of repository-local skills from
`.agents/skills/` and `.claude/skills/`, while keeping each skill name exactly
once and preserving the selected source package byte-for-byte.

## 2. Hard constraints

| # | Constraint |

| C1 | Do not modify `.agents/skills/` or `.claude/skills/`. |
| C2 | For names present in both sources, use the tool-neutral `.agents/skills/` package; use `.claude/skills/` only for names absent from `.agents/skills/`. |
| C3 | Create no duplicate skill directory names under `.codex/skills/`. |
| C4 | Match neighboring license headers and make no source-code or layout changes outside the complete touch list. |

## 3. Scope & glossary

**In:** repository-local skill packages and the required process records.
**Out:** global Codex skill installation, rewriting skill contents, agent
definitions, source code, build manifests, and unrelated knowledge documents.

- **Duplicate** — a second target package with the same skill directory name.
- **Selected source** — `.agents/skills/<name>` for overlapping names, otherwise
  `.claude/skills/<name>`.

## 4. Behaviour

The target is `.codex/skills/<name>/`. The target name set must equal the sorted
union of the two source name sets. Each target file must have the same SHA-256
hash as the corresponding file in its selected source package. No symlinks,
renames, adapters, or content normalization are permitted.
